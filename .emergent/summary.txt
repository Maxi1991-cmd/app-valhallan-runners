The user, a running coach, requested a mobile application to manage their athletes. The core features include a dual-role system (Coach/Athlete), program creation, workout tracking, a notification system, and a subscription model.

1.  **Coach Freemium Model**:
    *   **Free Plan**: Manage 1 athlete for free. There is no free trial period.
    *   **Premium Plan**: Manage unlimited athletes, unlocked via a real Stripe subscription (monthly or annual options).
2.  **Athlete Access**: Athlete accounts are always free.
3.  **Payment Log**: The Payments section in an athlete's profile is a private log for the coach to record payments received externally (e.g., cash, bank transfer). This log should allow for deleting entries.
4.  **Admin User**: The user's primary account () should have permanent, non-expiring premium access.
5.  **Notifications**: The system should send payment reminder notifications for athletes' private payments. The logic is: a first reminder 10 days before the due date of the most recent *unpaid* payment, then daily reminders from 3 days before until the due date. When a coach records a new payment, old reminders for that athlete should be cleared.
6.  **Deep Linking**: After a successful Stripe payment, the user should be automatically redirected back into the mobile app to their dashboard.
